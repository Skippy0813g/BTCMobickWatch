# Reporting a security vulnerability

## Please do not open a public issue

A vulnerability described in a public issue reaches attackers before it is fixed.
Report it privately instead:

**skippy0813w@gmail.com**

## What to include

- The conditions it occurs under, and how to reproduce it
- App version and Android version
- What an attacker would be able to do

When attaching logs or screenshots, **mask your own addresses, balances, and other
personal data.**

## What happens next

This is a solo project, so an immediate response may not be possible. You will get a
reply once the report has been reviewed, and any fix is noted in the release notes.
You can be credited by name or stay anonymous, whichever you prefer.
